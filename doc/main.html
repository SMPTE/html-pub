Authoring SMPTE documents consist of two aspects:

  • structure and syntax of the document; and
  • managing revisions using GitHub.

This Administrative Guideline describes the process for auhtoring SMPTE documents using HTML.

  • HTML Standard, Living Standard. https://html.spec.whatwg.org/multipage/

Directory layout

A document consists of multiple files arranged in the following directory structure:

  • doc/main.html: main element of the document (see ).
  • doc/media/: directory containing media referenced by the main element (see ).
  • doc/snippets/: directory containing snippets embedded in the main element (see ).
  • tooling/: directory consisting of the tools necessary to render SMPTE documents (see ).
  • .smpte-build.json: JSON file used to configure the HTML rendering process (see ).
  • .gitignore: Prevents build and operating system artifacts from being tracked by Git.
  • .github/workflows/main.yml: Allows automated HTML rendering and redlining in GitHub (see ).

head element

General

The head element shall contain the following:

  • itemscope attribute equal to itemscope;
  • itemtype attribute equal to http://smpte.org/standards/documents;
  • <meta charset="utf-8" />;
  • <meta http-equiv="x-ua-compatible" content="ie=edge" />;
  • <meta name="viewport" content="width=device-width, initial-scale=1" />
  • a script element whose src attribute links to the smpte.js module in the tooling directory;
  • a meta element that conforms to ;
  • a meta element that conforms to ;
  • a meta element that conforms to ;
  • a meta element that conforms to , if the document is published;

The title element shall be equal to the title of the document, without any document number.

pubType

The element shall have an itemprop attribute equal to pubType.

The element shall have a content attribute equal to one of the following values:

  • AG, if the document is an Administrative Guideline.

pubNumber

The element shall have an itemprop attribute equal to pubNumber.

The element shall have a content attribute equal to the document number.

pubState

The element shall have an itemprop attribute equal to pubState.

The element shall have a content attribute equal to one of the following values:

  • draft, if the document is a draft.
  • pub, if the document is published.

pubDateTime

The element shall have an itemprop attribute equal to pubDateTime.

The element shall have a content attribute that conforms to the pattern YYYY-MM-DD, where YYYY, MM and DD are the year, month and day of the publication date of the document.

body element

Overall structure

The body element shall consist of an ordered sequence of section elements, as specified at .

Ordered sequence of section elements comprising the body element.
Cardinality Section
0..1 introduction section
1 scope section
0..1 conformance section
0..1 normative references section
0..1 terms and definitions section
0..* prose section
0..* annex section
0..1 bibliography section

introduction section

The introduction section contains the introduction of the document.



        
The id attribute shall be present and equal to sec-introduction.


        
The heading will be generated automatically.

      


      

        
Scope


        
The scope section contains the scope of the document.


        
  <section id="sec-scope">
    <p>This is the scope.</p>
  </section>


        
The id attribute shall be present and equal to sec-scope.


        
The heading will be generated automatically.

      


      

        
Conformance


        
The conformance section contains conformance requirements and definitions, beyond the usual SMPTE conformance
        statements.


  
<section id="sec-conformance">
    <p>These are additional conformance requirements and definitions.</p>
  </section>
  


        
The id attribute shall be present and equal to sec-conformance.


        
The heading will be generated automatically.

      


      

        
Normative references


        
The normative references section contains a single unordered list ul where each element
        li is a normative reference.



<section id="sec-normative-references">
  <ul>
    <li>
      <cite id="bib-ietf-rfc-3986">IETF RFC 3986</cite>, Uniform Resource Identifiers (URI): Generic Syntax,
      T. Berners-Lee, et al., January 2005. <a>http://www.ietf.org/rfc/rfc3986.txt</a>
    </li>
  </ul>
</section>


        
The id attribute shall be present on the section element and equal to
        sec-normative-references.


        
The heading and boilerplate of the section will be generated automatically.


        
Each li element shall contain one cite element that contains the text that will be used when
        citing the reference in the prose.


        
Each li element may contain one a element that contains a location where the reference can be
        retrieved.


        
The id attribute shall be present on each cite element, and is used to
        link the reference from the prose as follows:


        
<a href="#bib-ietf-rfc-3986"></a>

      


      

        
Terms and definitions


        
The terms and definitions section contains terms that are not defined within the clauses of the document
        itself.


<section id="sec-terms-and-definitions">
  <ul id="terms-ext-defs">
    <li><a href="#bib-ietf-rfc-8259"></a></li>
  </ul>
  <dl id="terms-int-defs">
    <dt><dfn>key number</dfn></dt>
    <dd>number that is printed with ink or exposed onto the film at the time of
    manufacture at regular intervals, typically one foot.</dd>
  </dl>
</section>


        
The id attribute of the section element shall be equal to
        sec-terms-and-definitions.


        
The heading of the section element is generated automatically.


        
The section contains of two optional elements:

        
    
          
  • an unordered list ul whose li elements each contain a link to a normative references whose
          definitions are included by reference; and
    • 
          
  • an optional definition list dl that contains individual definitions.
    • 
        


        
The id attribute of ul element shall be equal to terms-ext-defs.


        
The id attribute of dl element shall be equal to terms-int-defs.


      


      

        
Prose


        
A prose section contains technical prose.



<section id="sec-clause-1">
  <h2>Clause heading</h2>
  <section id="sec-subclause-1">
    <h3>Heading of subclause 1</h3>
    <p>Some technical content</p>
  </section>
  <section id="sec-subclause-2">
    <h3>Heading of subclause 2</h3>
    <p>Some technical content</p>
  </section>
</section>



        
The id attribute shall be present.


        
A section heading element of the appropriate level shall be present, starting with level 2 (h2) for
        all prose sections that are children of the body element.


        
A prose section may contain other prose sections. If any child of a prose section is a prose section,
        then all children shall be prose sections.


        
The class attribute of a prose section shall not contain the annex class.


        
A prose section will be automatically numbered.

      


      

        
Annex


        
An annex section contains a technical annex.



<section class="annex" id="sec-annex-1">
  <h2>Annex heading</h2>
  <section id="sec-subclause-1">
    <h3>Heading of subclause 1</h3>
    <p>Some technical content</p>
  </section>
  <section id="sec-subclause-2">
    <h3>Heading of subclause 2</h3>
    <p>Some technical content</p>
  </section>
</section>



        
The requirements for an annex section are the same as those of a prose section, with the exception that the
        class attribute of an annex section shall contain the annex class.

      


      

        
Bibliography


        
The bibliography section contains a single unordered list ul where each element li
        is an informative reference.



<section id="sec-bibliography">
  <ul>
    <li>
      <cite id="bib-draft-bhutton-json-schema-00">IETF draft-bhutton-json-schema-00</cite>, Wright, A., H. Andrews; B. Hutton,
      G. Dennis, "JSON Schema: A Media Type for Describing JSON Documents", Work in Progress, December 8, 2020.
      <a>https://datatracker.ietf.org/doc/html/draft-bhutton-json-schema-00</a></li>
    </li>
  </ul>
</section>


        
The id attribute shall be present on the section element and equal to
        sec-normative-references.


        
The heading and boilerplate of the section will be generated automatically.


        
Each li element shall contain one cite element that contains the text that will be used when
        citing the reference in the prose.


        
Each li element may contain one a element that contains a location where the reference can be
        retrieved.


        
The id attribute shall be present on each cite element, and is used to
        link the reference from the prose as follows:


        
<a href="#bib-draft-bhutton-json-schema-00"></a>

Definitions


        
A term can be defined anywhere in the main element using the dfn element:


        
<dfn data-lt="alternate1|alternate2">term</dfn>


        

          The optional data-lt attribute specify alternate forms of the term, each separated by a | character,
          that can be used to link to the definition.
        


        
An optional id attribute can be specified; otherwise one will be generated automatically.


        The definition above can be referred to using one of the following forms:

        
    
          
  • <a>term</a>
    • 
          
  • <a>alternate1</a>
    • 
          
  • <a>alternate2</a>
    •
