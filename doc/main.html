Authoring SMPTE documents consist of two aspects:

  • structure and syntax of the document; and
  • managing revisions using GitHub.

This Administrative Guideline describes the process for auhtoring SMPTE documents using HTML.

  • HTML Standard, Living Standard. https://html.spec.whatwg.org/multipage/

Directory layout

A document consists of multiple files arranged in the following directory structure:

  • doc/main.html: main element of the document (see ).
  • doc/media/: directory containing media referenced by the main element (see ).
  • doc/snippets/: directory containing snippets embedded in the main element (see ).
  • tooling/: directory consisting of the tools necessary to render SMPTE documents (see ).
  • .smpte-build.json: JSON file used to configure the HTML rendering process (see ).
  • .gitignore: Prevents build and operating system artifacts from being tracked by Git.
  • .github/workflows/main.yml: Allows automated HTML rendering and redlining in GitHub (see ).

head element

General

The head element shall contain the following:

  • itemscope attribute equal to itemscope;
  • itemtype attribute equal to http://smpte.org/standards/documents;
  • <meta charset="utf-8" />;
  • <meta http-equiv="x-ua-compatible" content="ie=edge" />;
  • <meta name="viewport" content="width=device-width, initial-scale=1" />
  • a script element whose src attribute links to the smpte.js module in the tooling directory;

  • a meta element that conforms to ;
  • a meta element that conforms to ;
  • a meta element that conforms to , if the document is published;

The title element shall be equal to the title of the document, without any document number.

pubType

The element shall have an itemprop attribute equal to pubType.

The element shall have a content attribute equal to one of the following values:

  • AG, if the document is an Administrative Guideline;
  • OM, if the document is an Operations Manual.

pubNumber

The element shall have an itemprop attribute equal to pubNumber.

The element shall have a content attribute equal to the document number.

pubState

The element shall have an itemprop attribute equal to pubState.

The element shall have a content attribute equal to one of the following values:

  • draft, if the document is a draft.
  • pub, if the document is published.

pubDateTime

The element shall have an itemprop attribute equal to pubDateTime.

The element shall have a content attribute that conforms to the pattern YYYY-MM-DD, where YYYY, MM and DD are the year, month and day of the publication date of the document.

effectiveDateTime

The element shall have an itemprop attribute equal to effectiveDateTime.

The element shall have a content attribute that conforms to the pattern YYYY-MM-DD, where YYYY, MM and DD are the year, month and day that the document becomes effective.

The element shall be present if and only if pubType is OM.

body element

Overall structure

The body element shall consist of an ordered sequence of section elements, as specified at .

Ordered sequence of section elements comprising the body element.
Section Number (informative) Section Name Mandatory, Conditional, or Optional
Unnumbered Table of contents Automatically generated

Unnumbered Introduction Optional

4 Terms and definitions Mandatory
5-* * (Prose section) Mandatory

Unnumbered Bibliography Conditional (if informative references present)

Forward section

The Foreword section contains SMPTE boilerplate infomation based on the document type (pubType).

It can also contain author-specified text, e.g. list of substantive changes since the last edition. Author-specified text is added as follows:

Below is an example:

<section id="sec-foreword">
  <p>This is the additional infomation relevant to the document.</p>
</section>

The id attribute shall be present on the section element and equal to sec-foreword

The absence of this section indicates that the author did not provide additional user text.

The heading will be generated automatically.

Introduction section

The Introduction section contains the introduction of the document.

Below is an example:

<section id="sec-introduction">
  <p>This is the introduction.</p>
</section>

The id attribute shall be present on the section element and equal to sec-introduction.

The absence of this section indicates that the author did not provide introductory text.

The heading will be generated automatically.

Scope section

The Scope section contains the scope of the document.

Below is an example:

<section id="sec-scope">
  <p>This is the scope.</p>
</section>

The id attribute shall be present on the section element and equal to sec-scope.

The heading will be generated automatically.

Conformance section

The Conformance section contains conformance requirements and definitions.

If the conformance section is present as follows, any text within will be added to usual SMPTE conformance boilerplate.

Below is an example:

<section id="sec-conformance">
  <p>These are additional conformance requirements and definitions.</p>
</section>

The id attribute shall be present on the section element and equal to sec-conformance.

The absence of this section indicates that the author did not provide additional user text.

The heading will be generated automatically.

Normative references section

The Normative references section contains a single unordered list ul where each element li is a normative reference.

If using normative references, the section shall be formatted as: 

<section id="sec-normative-references">
  <ul>
    <li>
      <cite id="bib-HTML-5">HTML Standard</cite>, Living Standard. 
        <a>https://html.spec.whatwg.org/multipage/</a></li>
    </li>
  </ul>
</section>

The id attribute shall be present on the section element and equal to sec-normative-references.

li requirements (in order as above example show):

  • Each li element shall contain one cite element, with value that contains the text that will be used when citing the reference in the prose.
  • The cite element shall contain an id attribute with value prefixed by bib-.
  • Each li element may contain additional title infomation.
  • Each li element may contain one a element that contains a location where the reference can be retrieved.

When using citations in the prose, an a element with href attribute value equal to the above cite element's id attribute shall be used. This will generate an auto link back to the cite element with its value (i.e. ) rendered on the page. Example as follows:

<a href="#bib-HTML-5"></a>

The absence of this section indicates that the author did not provide normative references.

The heading and boilerplate of the section will be generated automatically.

Terms and definitions section

The Terms and definitions section contains terms that are not defined within the clauses of the document itself.

If terms are required to be defined, the section shall be formatted as: 

<section id="sec-terms-and-definitions">
  <ul id="terms-ext-defs">
    <li><a href="#bib-HTML-5"></a></li>
  </ul>
  <dl id="terms-int-defs">
    <dt><dfn>key number</dfn></dt>
    <dd>number that is printed with ink or exposed onto the film at the time of
    manufacture at regular intervals, typically one foot.</dd>
  </dl>
</section>

The id attribute of the section element shall be equal to sec-terms-and-definitions.

The section contains of two optional elements:

  • External definitions - shall be present as an unordered list ul
    • The id attribute of ul element shall be equal to terms-ext-defs.
    • Each li element contains a citation link to a normative reference (see ), whose definitions are included by reference
  • Internal definitions - shall be present in a definition list dl that contains individual definitions
    • The id attribute of dl element shall be equal to terms-int-defs.
    • Requirements for each term/definition pair:
      • Term - shall be present as a dt element, and must be wrapped in a dfn element.
      • Definition - shall be present as a dd element

The absence of this section indicates that the author did not provide definitions to terms.

The heading and boilerplate of the section will be generated automatically.

Prose section(s)

A Prose section contains technical prose.

As Prose sections are needed, the sections shall be formatted as: 

<section id="sec-prose-clause">
  <h2>Prose Clause</h2>
  <section id="sec-prose-sub-clause">
    <h3>Prose Sub Clause</h3>
    <p>Some technical content</p>
  </section>
  <section id="sec-next-prose-sub-clause">
    <h3>Next Prose Sub Clause</h3>
    <p>Some technical content</p>
  </section>
</section>

The id attribute shall be present, the value prefixed with sec-.

A section heading element of the appropriate level shall be present, starting with level 2 (h2) for all Prose sections that are children of the body element.

A Prose section may contain other nested Prose sections. If any child of a Prose section is a Prose section, then all children shall be Prose sections.

The class attribute of a Prose section shall not contain the annex class.

Avoid using letters and/or numbers for either Prose section id attribute and/or heading element value, as Prose sections will be automatically numbered based on placement on page.

Annex section(s)

An Annex section contains a technical annex.

<section class="annex" id="sec-addtional-info">
  <h2>Additional Info</h2>
  <section id="sec-addtional-info-sub-section">
    <h3>Additional Info Sub Section</h3>
    <p>Some technical content</p>
  </section>
  <section id="sec-addtional-info-sub-section-more">
    <h3>Additional Info Sub Section More</h3>
    <p>Some technical content</p>
  </section>
</section>

The requirements for an Annex section are the same as those of a Prose section, with the exception that the Annex section shall contain the class attribute with value of annex.

Avoid using letters and/or numbers, or the word annex for either Annex section id attribute and/or heading element value, as Annex sections will be automatically numbered based on placement on page and class.

Bibliography section

The Bibliography section contains a single unordered list ul where each element li is a informative reference.

If using informative references, the section shall be formatted as: 

<section id="sec-bibliography">
  <ul>
    <li>
      <li><cite id="bib-iso-directives-part2">ISO/IEC Directives, Part 2</cite>, Principles and rules for the structure and drafting of ISO and IEC documents (Ninth edition, 2021)
      <a>https://www.iso.org/sites/directives/current/part2/index.xhtml</a></li>
    </li>
  </ul>
</section>

The id attribute shall be present on the section element and equal to sec-bibliography.

The requirements for the Bibliography li are the same as those of Normative references.

When using citations in the prose, an a element with href attribute value equal to the above cite element's id attribute shall be used. This will generate an auto link back to the cite element with its value (i.e. ) rendered on the page. Example as follows:

<a href="#bib-iso-directives-part2"></a>

The absence of this section indicates that the author did not provide informative references.

The heading of the section will be generated automatically.

Other elements

Section linking

When using references to an internal section anywhere on the page, an a element with href attribute value equal to the section's id attribute shall be used. This will generate an auto link back to the section element with its section number/letter (i.e. - this section) rendered on the page. Example as follows:

<a href="#sec-prose-sub-clause"></a>

External links

Links to external resources shall be written by wrapping the URL in a a element as follows:

<a>https://www.iso.org/sites/directives/current/part2/index.xhtml</a>

An href attribute will automatically be generated.

Definitions

A term can be defined anywhere in the main element using the dfn element:

<dfn data-lt="alternate1|alternate2">term</dfn>

The optional data-lt attribute specify alternate forms of the term, each separated by a | character, that can be used to non-obtrusively link to the definition.

An optional id attribute can be specified; otherwise one will be generated automatically.

The definition above can be referred to in the prose using one of the following forms:

<a>term</a>
<a>alternate1</a>
<a>alternate2</a>

results in:

term alternate1 alternate2

Generic dl usage

dl elements may be used with the following requirement:

  • no class formatting or b applied to dt
  • at least one dd shall be present for each dt

For example:

<dl>
  <dt>imperative forms<dt>
  <dd>"see"</dd>
  <dt>non-imperative forms</dt>
  <dd>"according to"</dd>
  <dd>"as defined in"</dd>
  <dd>"as specified in"</dd>
  <dd>"details as given in"</dd>
  <dd>"in accordance with"</dd>
</dl>

dl elements will automatically be formatted.

Figures

Figures can be inserted by using a figure element with the following requirements:

  • id attribute shall be present, the value prefixed with figure-
  • shall contain one img element, with class attribute present with value figure
  • shall contain one figcaption element BELOW, with value that describes the figure

For example:

<figure id="figure-sample-image">
  <img class="figure" src="media/sample.png">
  <figcaption>Example of a figure</figcaption>
</figure>

results in:

When using references to a figure anywhere on the page, an a element with href attribute value equal to the figure's id attribute shall be used. This will generate an auto link to the figure element with Figure and its numerical value (i.e. ) rendered on the page. Example as follows:

<a href="#figure-sample-image"></a>

Tables

Tables can be inserted by using a table element with the following requirements:

  • id attribute shall be present, the value prefixed with tab-
  • shall contain one caption element ABOVE, with value that describes the table
  • shall contain one thead element, containing one tr element, with th element(s) as needed for each column header
  • shall contain one tbody element, with tr element(s) as needed for each row

For example:

<table id="tab-sample-tabledata">
  <caption>Sample Table</caption>
  <thead>
    <tr>
      <th>Sample Number</th>
      <th>Sample Name</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>0001</td>
      <td>Name 01</td>
    </tr>
    <tr>
      <td>0002</td>
      <td>Name 02</td>
    </tr>
  </tbody>
</table>

results in:

Sample Table
Sample Number Sample Name
0001 Name 01
0002 Name 02

When using references to a figure anywhere on the page, an a element with href attribute value equal to the tables's id attribute shall be used. This will generate an auto link to the table element with Table and its numerical value (i.e. ) rendered on the page. Example as follows:

<a href="#tab-sample-tabledata"></a>

Notes


        
Notes can be inserted by using p element with a class attribute value of note


        
Notes will automatically be formatted, prepended with Note - , and numbered (or not) based on the section they are in and number of notes per section as per .

        
For example:



<p class="note">
  The license portion of Annex A is verbatim the BSD-3-Clause license
  available at <a>https://spdx.org/licenses/BSD-3-Clause.html</a>.
</p>


      
        
results in:


        

          The license portion of Annex A is verbatim the BSD-3-Clause license
          available at https://spdx.org/licenses/BSD-3-Clause.html.
        



      


      

        
Examples


        
Examples can be inserted by using p element with a class attribute value of example.


        
Examples will automatically be formatted and prepended with EXAMPLE:.

        
For example:



<p class="example">The number <i>0.0003</i> can be written 
<i>3 × 10−4</i> but is never written <i>3e−4</i>.</p>



        
results in:


        
The number 0.0003 can be written 
          3 × 10−4 but is never written 3e−4.
